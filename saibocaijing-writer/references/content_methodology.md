# 赛博财经内容方法论

这份文档讲的是「写什么」和「怎么找到该写的角度」，是 SKILL.md 的补充。

## 1. 选题交集模型

一个好的赛博财经选题 = **科技/AI前沿事件** × **财经化钩子** × **时效性**

三者缺一就会滑向其他风格：
- 缺财经化钩子 → 变成科技自媒体（讲参数、讲效果）
- 缺AI/前沿 → 变成传统财经号（讲传统行业、讲基金）
- 缺时效性 → 变成科普讲座（没人看）

**好选题的例子**：
- 一家 AI 公司的裁员潮（AI事件 × 资本/人力成本 × 正在发生）
- 内存价格暴涨（科技硬件 × 成本转嫁普通人 × 近 3 个月数据）
- 黄仁勋访华 × 老黄的公关姿态 × 国产替代节点（人物行动 × 股价博弈 × 刚发生）
- 美国巨头抢核电（算力 × 能源产业链 × 2026 分水岭）

**差选题的例子**（单独出现就要谨慎）：
- "Claude Sonnet 4.7 发布了"（纯产品测评，没有财经钩）
- "ChatGPT 的 10 种 Prompt 技巧"（工具分享，不是赛博财经选题）
- "AI 的发展史"（没有时效性）

## 2. 四种稿件原型详解

### 产业博弈型

**核心结构**：两方或多方对决 → 各自手里的牌 → 正在动的棋 → 下一步走向

**适用场景**：
- 中美科技博弈（芯片、算力、模型）
- 硅谷内战（OpenAI vs Anthropic、英伟达 vs 反英伟达联盟）
- 资本博弈（股东 vs 员工、AI 大厂 vs 下游软件公司）

**结构模板**：
```
一、X 的焦虑（描述某方的处境和行动）
二、Y 的反击（描述另一方的应对）
三、第三方变量（可能是技术、可能是政策、可能是黑马）
四、双极世界（升华到文明对决）
```

**代表样稿**：《2026，美国AI没电了》《硅谷72小时》《黄仁勋访华》

**写作要点**：
- 每一方的"牌"要具体（产能数字、融资规模、人事动向）
- 避免简单的"好人vs坏人"，要呈现双方逻辑都合理但撞车
- 升华要落到"这是两种 xxx 的对决"这种定性判断

### 事件解密型

**核心结构**：表面现象 → 为什么 → 背后机制 → 长远影响

**适用场景**：
- 一个让人困惑的财经现象（裁员利好股价、内存比黄金涨得快）
- 一个刷屏但大家似懂非懂的事件（AI 投毒、三宫格刷屏）
- 一个需要用财经逻辑解释的反直觉现象

**结构模板**：
```
一、一个事件（铺陈事实）
二、资本/市场为什么 xxx？（揭开第一层）
三、xxx 真的能 xxx 吗？（质疑/深入）
四、未来会怎样（拉到大背景）
```

**代表样稿**：《AI 裁员剪辑稿》《一盒内存条，能买上海一套房》

**写作要点**：
- 第一幕要把"反直觉"的点完整铺开，让读者心里打问号
- 第二三幕层层揭开，每一层都给新信息
- 每一层都要有具体数据支撑

### 人物/地标切入型

**核心结构**：一个具体画面 → 这个画面背后的逻辑 → 这个逻辑揭示的行业动向

**适用场景**：
- 大佬的反常动作（黄仁勋买菜、马斯克访谈）
- 关键地标的探访（智算中心、数据中心、核电站）
- 重要会议/发布会现场

**结构模板**：
```
【开场：极具体画面】黄仁勋在菜市场买水果、冷却塔的轰鸣声
一、这个人/地方为什么在这里？
二、他/它代表的是什么？
三、背后的产业/格局正在怎么变
四、下一步会怎样
```

**代表样稿**：《黄仁勋访华》《AI 是重工业（成都智算中心）》

**写作要点**：
- 开场的画面必须是"反差感"强的，不是官方通稿式的
- 画面 → 行业的跳跃要自然，不要硬拽
- 最好有第一手观察或独家细节

### 未来预言型

**核心结构**：当前一个节点事件 → 推演近期走向 → 推演中期格局 → 给出时间坐标

**适用场景**：
- 年初/年末的趋势预测
- 某个关键事件标志着范式转变
- 重要论文/报告的解读

**结构模板**：
```
【开场：一个正在发生的暴击事件】
一、短期（几个月内）会发生什么
二、中期（1-3 年）格局变化
三、长期（3-5 年）的大趋势
四、我们正站在什么位置
```

**代表样稿**：《2026，美国AI没电了》、《2028全球智能危机》（被引用的内容）

**写作要点**：
- 所有预言必须有当下数据锚定，不能空推
- 时间坐标要具体（"2027 年""2030 年"不是"未来"）
- 警惕"大饼"化，不要只喊口号

## 3. 财经化钩子工作法

拿到一个纯科技素材后，用下面的工具箱把它"翻译"成赛博财经的视角：

### 资本钩（找钱在哪）
- **融资视角**：谁投了谁？投资方是战略投资还是财务投资？这家公司的估值逻辑是什么？
- **股价视角**：这件事发生后相关公司股价怎么动了？为什么资本这么反应？
- **市值视角**：这件事让谁的市值增减了多少？粗算每项行动创造/摧毁的账面价值？

### 产业链钩（找上下游）
- **上游**：这件事依赖什么？谁是供应商？供应商会被怎么影响？
- **下游**：这件事产出什么？谁在消费？消费者会被怎么影响？
- **对手**：这件事的直接/间接竞争对手是谁？他们会怎么反应？

### 成本钩（找谁买单）
- **直接成本**：价格涨了吗？谁付的？
- **间接成本**：订阅涨价？服务缩水？基础设施成本转嫁？
- **机会成本**：某个产能/资源被抢走了，另一端被挤压了？

### 博弈钩（找对决）
- **地缘**：中美、欧美、跨洋产业链
- **公司**：巨头间的联盟/内战
- **阶层**：资本 vs 员工、科技精英 vs 大众、硅谷 vs 华盛顿
- **代际**：老巨头 vs 新玩家、旧技术范式 vs 新范式

### 拐点钩（找时间坐标）
- **为什么是现在**：这件事为什么在这个时点发生？
- **和历史对照**：这像历史上的哪件事？（AlexNet 时刻、iPhone 时刻、奥本海默时刻）
- **下一个节点**：下一个值得关注的时间点是什么？（产品发布、财报、政策）

**使用方法**：拿到一个素材，至少用上面 2-3 个钩子去翻找角度，选其中最有冲击力的一两个作为主叙事线。

## 4. 参考资料的正确使用姿势

用户提供参考资料时（新闻链接、财报、PDF、访谈），按下面的流程处理：

**Step 1：吃透原始材料**
- 把材料里所有具体数字、时间、人名、公司名、产品名圈出来
- 找出 3-5 个"反差感"最强的点（最大的数字、最反直觉的事实、最戏剧化的细节）

**Step 2：选定财经钩**
- 用第 3 节的五类钩子筛选，找到这个素材最强的 1-2 个钩子
- 如果钩子很弱，主动跟用户说"这个素材的财经钩比较弱，我建议从 xxx 角度切，或者补充 xxx 资料"

**Step 3：选定原型**
- 按第 2 节选一个最适合的原型
- 在大脑里跑一遍该原型的结构模板，看素材里的信息能不能填满每一幕

**Step 4：补充外围数据**
- 如果某个数据需要对比才有冲击力，主动找参照（10 万英里 = 绕赤道 4 圈）
- 如果某个公司/产品背景读者不熟，准备一句话介绍
- 如果是国外事件，准备好中国对应的情况用来对比

**Step 5：起标题和 Hook**
- 在动笔正文前，先把 5 个标题和 5 个 Hook 写出来
- 标题和 Hook 是整期稿的"导航仪"，它们定了主叙事线之后，正文会跟得上

## 5. 画面/报告提示怎么标（极简版）

后期老师会自己发挥镜头、转场、b-roll、字幕、音效。写稿人**只标两件事**：

### 只用两种标注

- `[画面:xxx]` — 这里需要**上一张具体的图/视觉素材**（人物街拍、产品实物、现场照片、地点场景、截图）
- `[报告:xxx]` — 这里需要**露出报告原文/文件截图/数据页**（财报某一页、研报封面、政策文件、发布会 PPT）

### 该标的 4 类位置

1. **重要人物首次出现**：`[画面:黄仁勋在北京某餐馆用餐的街拍]`
2. **具体产品/实物/地标**：`[画面:一盒 256GB DDR5 内存条实物]`、`[画面:三里岛核电站冷却塔]`
3. **报告/研报/财报被直接引用**：`[报告:斯坦福 2026 AI Index 报告封面]`、`[报告:Block 最新季度财报首页]`
4. **需要视觉冲击的反差时刻**：`[画面:一张上海 70 平房源挂牌图]`（呼应"一盒内存能买一套房"）

### 不要再写的标注

`【压字:xxx】`、`【人名条:xxx】`、`【划重点:xxx】`、`【素材:xxx】`、`【数据图:xxx】`、`【录屏:xxx】`、`【镜头:xxx】`、`【转场:xxx】`、`【b-roll:xxx】`、`【音效:xxx】`——全部交给后期,**写稿人不标**。

### 格式规范

- 用**英文方括号** `[画面:xxx]` / `[报告:xxx]`,紧跟在相关口播句之后
- 冒号后写**具体到可执行**的描述(后期看到能马上知道找什么素材)
- 一期稿 `[画面]` 5-10 处,`[报告]` 0-3 处
- 不要句句标。只在"这里不上图就抽象"的位置才标

## 6. 改稿的时候重点看什么

第一稿写完跑完 L1/L2/L3 自检后，还要用"用户视角"再读一遍。重点检查：

**读到第 10 秒（Hook + 品牌开场）**：
- 如果我在刷抖音，我会不会划走？
- 第一句有没有具体到让人停一秒？

**读到第 1 分钟（第一幕）**：
- 这件事有多大、跟我什么关系，讲清楚了吗？
- 有没有至少一个让我"哦？"的数据或细节？

**读完全程**：
- 我看完之后，能记住几个具体数字/具体名字？（至少要能记住 3 个）
- 我能给朋友复述这件事是怎么回事吗？
- 我看完有没有想转发的冲动？哪句话会让我想截图？

如果这三层里有任何一层答"不会"，回去改。

## 7. 什么情况下跟用户对话、不直接开写

这些情况主动找用户对齐，不要硬写：

1. **素材只有一个方向，没有具体资料**："写一期英伟达"——问用户想切哪个角度
2. **素材财经钩很弱**：比如用户丢了一个纯产品测评，没有融资/股价/产业链信息——问用户想补什么资料
3. **素材里有互相矛盾的信息**：两个数据不一致，或者事件时间线模糊——问清楚
4. **素材里的关键数字已经过时**：比如用户给的是半年前的财报，但刚发了新财报——确认用哪版
5. **选题可能有合规/舆情风险**：涉及敏感地缘、可能惹麻烦的个人攻击——先问用户风险承受度

对齐的问法要具体、不要泛泛："你想切 xxx 还是 xxx 角度？"、"有没有 xxx 数据？"、"你自己最想抛的吐槽点或观点是什么？"

## 8. 产出前的最终审查清单

每次交付前，对着这份清单再过一遍：

- [ ] 顶部有【5 个抖音爆款标题】+【5 个 Hook 勾子】
- [ ] 5 个标题公式不重样
- [ ] 5 个 Hook 类型不重样
- [ ] 开场是 Hook → 悬念 → 品牌开场
- [ ] 主体 3-4 幕，每幕有悬念式小标题
- [ ] 全文有 ≥5 处具体数字
- [ ] 所有公司/人物/产品都有具体名字
- [ ] ≥1 处产业链视角
- [ ] ≥1 处普通人锚点
- [ ] 升华结尾拉到文明/历史/产业坐标
- [ ] 固定品牌尾
- [ ] 🦊🐟 每幕切换 ≥3 次
- [ ] 标注只用 `[画面:xxx]` 和 `[报告:xxx]` 两种,无多余的镜头/转场/压字/b-roll/音效标注
- [ ] 总字数 2000-3000
- [ ] 质检报告一并输出
