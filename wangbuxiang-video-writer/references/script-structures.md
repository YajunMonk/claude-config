# 口播结构与时长

## 时长判断

- 约 1 分钟：一个观点、一个小技巧、一个反直觉提醒。
- 约 2 分钟：一个真实案例 + 一个方法。
- 约 3 分钟：一个产品体验、一个工作流复盘、一个学习路径。
- 约 4 分钟：复杂主题、争议话题、需要铺垫背景和边界的内容。

默认不要为了完整而拉长。主题简单就短讲。

## 四种常用结构

### 1. 真实案例型

适合工作提效、生活应用、录音笔记。

结构：真实场景 -> 遇到的问题 -> AI 怎么介入 -> 前后变化 -> 普通人判断。

### 2. 产品体验型

适合 AI 产品观察。

结构：我最近用了什么 -> 一个具体细节 -> 为什么这个细节重要 -> 适合谁/不适合谁 -> 普通人能学什么。

### 3. 学习方法型

适合普通人学 AI。

结构：常见误区 -> 正确入口 -> 一个可执行练习 -> 怎么判断有没有进步 -> 互动提问。

### 4. 教育支线型

适合父亲视角和未来教育。

结构：一个家庭/孩子/学习场景 -> 父母容易误判的地方 -> AI 真正改变的不是工具而是能力 -> 给家长一个克制建议。

## 成稿要求

- 一条视频只讲一个核心点。
- 前 3 秒必须让人知道“这和我有关”。
- 中段必须有可见动作：演示、前后对比、步骤、案例，不要只讲道理。
- 结尾不要总结大会，要留下一个普通人能记住的判断。
- 口播句子短一点，但不要机械短句。读起来要像人在想、人在说。

## 字数参考

- 1 分钟：250-350 字。
- 2 分钟：450-650 字。
- 3 分钟：700-950 字。
- 4 分钟：950-1250 字。

字数不是硬指标。顺口和完整优先。
