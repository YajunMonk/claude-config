# 王不响公众号内容方法论

本文是 `wangbuxiang-edu-writer` 的选题和结构参考。默认服务公众号长文，不服务短视频口播。

## 当前定位

王不响是一个真实做 AI 的朋友，把普通人怎么学 AI、怎么用 AI 提升工作产出、怎么把 AI 用进生活，讲得真实、具体、听得下去。

默认内容比例：

- AI 学习 / 工作提效 / 生活应用：70%
- AI 教育 / 父亲视角 / 未来教育观察：30%

教育支线可以成为差异化，但不能把账号写回单纯家长号、少儿 AI 号或未来教育号。

## 好选题标准

一个好选题通常来自三者交集：

- 作者真实接触过，或能通过公开材料扎实验证。
- 普通人真的关心，能省时间、少踩坑、提升判断，或看见一个新机会。
- 当下有时间节点，来自热点、产品更新、平台变化、工作场景或身边人的真实困惑。

对标爆文时，再加一层筛选：

- 外部性强：硬现场来自公开市场、平台、评论区、产品页、榜单、价格、服务或可核查案例，而不是只来自内部思考。
- 反差强：读者一眼能看出“这事有点不对劲”，比如 100000+ prompts 卖几美元、工具清单越收藏越不会用。
- 普通人强相关：不懂技术的人也能代入，比如学 AI、找工具、写方案、做副业、怕落后、被教程淹没。
- 能升维但不空：能从小现场讲到信息差、学习方式、工作流或普通人处境，最后还要落回今天能做什么。

提示词类选题要多看一层：它卖的是“角色壳”，还是帮普通人补上了任务、背景、约束、样例和成功标准。前者解决入口焦虑，后者才可能真正改善输出。

动笔前用 HKR 做快速判断：

- H，是否有悬念、反差、具体问题或好奇心。
- K，是否有信息量，读者看完能学到东西。
- R，是否能让读者觉得这跟我有关。

三项都强，可以写深文。两项成立，可以写。只成立一项，先换角度。

如果目标是“比范文更好”，不要只看观点对不对，先看硬现场够不够强。硬现场弱，文章上限通常不高。

## 五个栏目

**普通人学 AI**
写给零基础或非技术背景的人。重点不是工具清单，而是如何开始、如何判断、如何形成自己的信息源和练习场。
常见选题：怎么从 0 理解大模型、为什么每天和 AI 聊天是低成本学习、不会代码的人怎么进入 AI、如何建立 AI 信息源。

**AI 工作实验室**
把 AI 用进真实工作产出。重点是工作流、前后变化和可复制动作。
常见选题：怎么把模糊想法聊成方案、AI 写文档真正改变什么、运营人的 AI 工作流、怎么用 AI 做复盘、普通职场人做自己的小工具。

**AI 生活应用**
把 AI 放进生活、学习和家庭。重点是普通人高频、低门槛、能持续使用的场景。
常见选题：用 AI 做阅读记录、整理生活困惑、旅行计划、长期目标拆解、家庭使用边界。

**AI 产品观察**
从产品细节讲普通人能学到什么。重点是体验、场景、边界和判断，不写成行业报告。
常见选题：一个好 AI 产品不一定要做大模型、低频需求为什么容易被 ChatGPT 吃掉、交互体验为什么重要、从一个功能看应用层机会。

**未来教育支线**
自然相关时再写。重点是 AI 时代的提问、阅读、判断、表达、创造，不是制造家长焦虑。
常见选题：孩子要不要学 AI、AI 写作业的边界、家庭 AI 使用协议、不要把 AI 变成新的鸡娃工具。

## 文章原型

**AI 实践调查型**
入口：一个荒诞现象、服务价格、平台乱象、工具热潮或真实需求。
推进：亲自搜索、下单、注册、询问、试用、对比、记录结果。
收尾：从个案回到普通人处境、信息差、安全、学习成本或时代变化。

**产品体验型**
入口：一个新产品、新功能或旧产品的新用法。
推进：具体任务 -> 使用过程 -> 结果 -> 翻车/惊喜 -> 适合谁。
收尾：给普通人的判断，值得用在哪里，不值得神化在哪里。

**普通人方法论型**
入口：一个真实卡点，比如学不会、用不上、收藏太多、产出太少。
推进：观点 -> 场景 -> 做法 -> 失败点 -> 最小行动。
收尾：让读者今天能做一件小事。

**实操教程型**
入口：一个明确产出或真实卡点，比如“我想让 Codex 帮我改一篇文章，但不知道从哪开始”。
推进：目标 -> 环境/入口 -> 第一次操作 -> 结果检查 -> 常见卡点 -> 下一步。
收尾：让读者完成一个最小闭环，而不是记住一堆命令。

**概念转译型**
入口：一个新手误解，比如“Agent 是不是更聪明的 ChatGPT”。
推进：真实困惑 -> 一个类比 -> 一个小实验或场景 -> 概念边界 -> 普通人判断。
收尾：把概念落成一个使用判断或练习动作。

**短视频脚本扩写型**
入口：把脚本里的强钩子改成真实入口，不照搬“前三秒抓人”的句式。
推进：核心判断 -> 场景补足 -> 过程/案例 -> 普通人处境 -> 边界和行动。
收尾：删掉关注、点赞、收藏类话术，改成公众号式回环、判断或一个今天能做的小动作。

**现象解读型**
入口：一个热点、一份名单、一条评论、一个平台变化。
推进：表面现象 -> 更深问题 -> 公开材料/案例 -> 个人判断。
收尾：把复杂问题翻译成普通人能用的判断。

**未来教育支线型**
入口：孩子、家庭、老师、课程、教育新闻或 AI 学习场景。
推进：具体场景 -> 家长/孩子真实处境 -> AI 介入边界 -> 能力判断。
收尾：回到好奇心、判断力、表达力、创造力等长期能力。

## 结构原则

不要先套模板，让内容决定结构。常用骨架是：

1. 具体入口，把读者拉进现场。
2. 第一层发现，说明这事为什么值得看。
3. 过程或案例，给读者可见动作。
4. 人物或普通人处境，把信息转成人。
5. 知识、历史、行业背景，像顺手掏出来，不像上课。
6. 判断，告诉读者这件事对普通人有什么用。
7. 回环收尾，回到开头的意象或问题。

每偏出去一段，都要用一句话扣回主线。扣主线句不需要漂亮，只要能让读者知道你还在讲同一件事。

## 选题来源

- 自己真实工作流：AIGC 内容、AI 投放、GEO、文档、复盘、运营流程。
- 公开产品和工具：ChatGPT、Claude、Codex、豆包、Kimi、AI 搜索、AI PPT、AI 浏览器。
- 普通人困惑：不会开始、工具太多、提示词没用、担心被淘汰、学了用不上。
- 短视频脚本：已有口播、分镜、短视频提纲、视频文案、直播切片观点。
- 生活场景：阅读、旅行、家庭计划、学习记录、情绪整理、亲子沟通。
- 未来教育：AI 写作业、孩子提问、家庭规则、学校变化。
- 热点材料：公司发布、产品更新、榜单、论文、政策、社交平台讨论。

涉及事实和新信息时先核查。没有核查就不要写成确定结论。
